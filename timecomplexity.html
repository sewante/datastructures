Time Complexity

Time complexity of an algorithm refers to the total time required by the program to run to completion. The time depends on the input, most often on the size of the input. Time complexity is most often expressed using the Big O notation Time Complexity is most commonly estimated by counting the number of elementary functions performed by the algorithm

Worst time complexity

worst-case Time complexity of an algorithm because that is the maximum time taken for any input size. Since the algorithm's performance varies with different input sizes, the worst case is considered to be efficient.

Calculating time complexity of an algorithm

Time Complexity is most commonly estimated by counting the number of basic operations performed by the algorithm

Big O notation

The common metric of calculating time complexity is the Big O notation. This removes all constant factors so that the running time can be estimated in relation to N, as N approaches infinity.

Consider the algorithms below

 1. statement;
Above we have a single statement. Its Time Complexity will be Constant. The running time of the statement will not change in relation to N. 
 1. for(i=0; i < N; i++) {
 2.	    statement;
 3. }
The time complexity for the above algorithm will be Linear. The running time of the loop is directly proportional to N. When N doubles, so does the running time. 
 1. for(i=0; i < N; i++) {
 2.     for(j=0; j < N;j++) { 
 3.         statement;
 4.     }
 5. }
This time, the time complexity for the above code will be Quadratic. The running time of the two loops is proportional to the square of N. When N doubles, the running time increases by N * N. 
 1. while(low <= high) {
 2. 	mid = (low + high) / 2;
 3. 	if (target < list[mid])
 4. 		high = mid - 1;
 5. 	else if (target > list[mid])
 6. 		low = mid + 1;
 7. 	else
 8. 	break;
 9. }
This is an algorithm to break a set of numbers into halves, to search a particular field(we will study this in detail later). Now, this algorithm will have a LogarithmicTime Complexity. The running time of the algorithm is proportional to the number of times N can be divided by 2(N is high-low here). This is because the algorithm divides the working area in half with each iteration. 
 1. void quicksort(int list[], int left, int right) {
 2. 	int pivot = partition(list, left, right);
 3. 	quicksort(list, left, pivot - 1);
 4. 	quicksort(list, pivot + 1, right);
 5. }
Taking the previous algorithm forward, above we have a small logic of Quick Sort(we will study this in detail later). Now in Quick Sort, we divide the list into halves every time, but we repeat the iteration N times(where N is the size of list). Hence time complexity will be N*log( N ). The running time consists of N loops (iterative or recursive) that are logarithmic, thus the algorithm is a combination of linear and logarithmic. NOTE : In general, doing something with every item in one dimension is linear, doing something with every item in two dimensions is quadratic, and dividing the working area in half is logarithmic.
